The Journey V1 → V10

10 iterations. Each one fixed a class of failure the previous one revealed.

V1 — "Lattes at €28,000"

Tesseract OCR. Garbage in, garbage out. Numbers from thermal paper noise, misread decimals, zero structure.

V2 — "Bananas When It Said Lemons"

Switched to Claude Vision. Hallucinations: confident, structured, and completely wrong. The model filled in nulls with plausible-looking data.

V3–V5 — Null-Output Hardening

11 hard rules enforcing "null over guess." tool_use structured output replaced "please return JSON." Added PaddleOCR cross-validation.

V6–V10 — Production Hardening

Circuit breaker, rate limiter, TOCTOU cost guard, IQR outlier removal, pack-size regex, multi-language support, 195 tests.

"V10 fixed bugs that never caused a crash—they just made the system quietly wrong."

Null Philosophy

A missing value is infinitely better than a hallucinated one. Downstream stages can handle nulls; they can't detect confident lies.

Cross-Validation

Asymmetric trust: PaddleOCR flags but never overrides Claude.

The Trust Model

Claude is the authority. PaddleOCR is the auditor.

  • PaddleOCR extracts raw text independently
  • Compare prices against Claude's line items
  • Disagreement → flag, don't replace
  • Math tie-breaker: if qty × price = total, trust that source

Resilience Patterns

Three patterns protecting against API failures and cost overruns.

Token-Bucket Rate Limiter

Classic token-bucket with a critical detail:

async with self._lock: if tokens >= 1.0: tokens -= 1.0 return wait = (1.0 - tokens) / rate # Sleep OUTSIDE the lock await asyncio.sleep(wait)

Bug found: V8 held the lock during sleep, blocking all coroutines.

Circuit Breaker

3-strike pattern with half-open recovery:

  • Closed: normal operation
  • Open: 3 consecutive failures → skip to Tesseract fallback
  • Half-open: after recovery_sec, allow one probe request

Prevents hammering a failing API while allowing automatic recovery.

TOCTOU Cost Guard

Atomic reservation prevents concurrent overshoot:

async with self._cost_lock: # Reserve BEFORE the call self.cost += 0.05 # ... make API call ... async with self._cost_lock: # Reconcile: actual - estimate self.cost += actual - 0.05

Without reservation, 8 concurrent workers could all pass the check.

  • Partial obstruction
  • Rotated/skewed scans

  • Missing decimal points

Honest Limitations

What this pipeline doesn't do — and why that's deliberate.

Static FX Rates

EUR/GBP/CHF rates are hardcoded constants. No live API. Fine for weekly batch runs; wrong for real-time pricing.

No Learning Loop

Corrections don't feed back. The override file is manual. A feedback loop would need human-in-the-loop UI.

Single-Image Only

One receipt = one image. Multi-page receipts or PDF invoices need stitching logic not yet built.

Tesseract Languages

Missing 3 language packs (Dutch, Portuguese, Italian). Tesseract fallback is EN/FR/DE/ES only.

Duplicates Downweighted

Detected duplicates are flagged and confidence-downgraded, but not removed. Downstream must filter.

Seasonal Pricing

No time-series awareness. A strawberry receipt from June and December get equal weight.

Key Lessons

"Confident hallucination is worse than noisy failure."
V2's Claude output was beautifully structured and completely wrong. Tesseract's garbage was at least honestly garbage. The danger isn't wrong data — it's wrong data that looks right.
"The biggest accuracy gain was a JSON file, not an algorithm."
overrides.json took mapping rate from 54% to 95%. No ML, no embeddings, no fine-tuning. Just a curated list of "Arabica Beans (1kg)" → "coffee_beans".
"The rate limiter held a lock while sleeping."
A concurrency bug that didn't crash — it just made every request sequential. The kind of bug that hides in load tests because "it works, just slowly."
"V10 fixed bugs that never caused a crash — they just made the system quietly wrong."
The scariest bugs aren't exceptions. They're silent data corruption: costs off by 5%, confidence scores that don't degrade, pack sizes parsed as unit prices.
